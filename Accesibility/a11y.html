ARIA Quick Revision

ARIA (Accessible Rich Internet Applications) adds extra meaning for assistive technologies when native HTML is not enough.

Rule of thumb: use semantic HTML first, add ARIA only when needed.

Most Important ARIA Attributes

Attribute Category Common Use Case
aria-label Labeling Provides a text label for elements with no visible text, like icon-only buttons (e.g., a "hamburger" menu icon).
aria-labelledby Relationship Links an element to another element's text to use as its label. Often used for modals to point to the heading.
aria-describedby Relationship Provides a detailed description by linking to a separate text block, like form field hints or error messages.
aria-expanded State Tells a screen reader if a collapsible element (like a dropdown or accordion) is currently open or closed.
aria-hidden State Removes an element (like a decorative icon) from the accessibility tree so it's ignored by screen readers.
aria-live Live Region Announces content updates that happen without a page reload, such as live search results or status alerts.
aria-current State Identifies the active page in a navigation list or the current step in a breadcrumb trail.
